# Infrastructure Architecture

## Overview

NeoForge uses a containerized microservices architecture deployed on a single Digital Ocean droplet, with Cloudflare CDN for edge caching and distribution.

## Architecture Diagram

```mermaid
graph TB
    subgraph Internet
        User((User))
        CF[Cloudflare CDN]
    end

    subgraph "Digital Ocean ($10 Droplet)"
        subgraph "Docker Containers"
            NGINX[NGINX Proxy]
            FE[Frontend<br/>Lit Components]
            API[FastAPI Backend]
            DB[(PostgreSQL)]
            CACHE[(Redis Cache)]
        end

        subgraph "Monitoring"
            PROM[Prometheus]
            NODE[Node Exporter]
        end
    end

    User --> CF
    CF --> NGINX
    NGINX --> FE
    NGINX --> API
    API --> DB
    API --> CACHE

    PROM --> NODE
    NODE --> FE
    NODE --> API
    NODE --> DB
    NODE --> CACHE
```

## Deployment Flow

```mermaid
sequenceDiagram
    participant Dev as Developer
    participant Git as GitHub
    participant CI as GitHub Actions
    participant DO as Digital Ocean
    participant CF as Cloudflare

    Dev->>Git: Push Code
    Git->>CI: Trigger Workflow
    CI->>CI: Run Tests
    CI->>CI: Build Images
    CI->>DO: Deploy Containers
    DO->>CF: Update Edge Cache
    CF->>Dev: Deployment Complete
```
